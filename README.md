# 🩺 Multimodal Medical Report Analyzer

An AI-powered web application that combines **Computer Vision** and **Natural Language Processing** to analyze Chest X-Ray images alongside clinical reports. The system detects visual anomalies using a fine-tuned **YOLOv8** model, cross-references them with parsed clinical text, and generates a patient-friendly summary via **Google Gemini**.

## 🧠 How It Works

The application follows a three-stage multimodal pipeline:

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────────┐
│  1. UPLOAD  │────▶│  2. AI ANALYSIS  │────▶│  3. REVIEW RESULTS  │
│  X-Ray +    │     │  YOLO + Gemini   │     │  Annotated Image +  │
│  Report     │     │  Vision + NLP    │     │  Patient Summary    │
└─────────────┘     └──────────────────┘     └─────────────────────┘
```

### Stage 1 — Upload
The user uploads two files:
- A **Chest X-Ray image** (PNG/JPG, up to 10MB)
- A **Clinical Report** (PDF or TXT)

### Stage 2 — AI Analysis
The backend processes both inputs through three specialized services:

1. **Vision Engine** — A fine-tuned YOLOv8 model runs inference on the X-Ray to detect and localize medical anomalies (e.g., Cardiomegaly, Aortic Enlargement, Pleural Effusion). Detections include bounding box coordinates and confidence scores.
2. **Parsing Layer** — The clinical report (PDF/TXT) is parsed using PyMuPDF to extract structured `Findings` and `Impression` sections via regex-based extraction.
3. **Multimodal Orchestration** — Both the vision detections and the parsed text are fed into a **LangChain prompt chain** powered by **Google Gemini (gemini-flash-latest)**. The LLM cross-references visual anomalies with clinical notes and generates a unified, patient-friendly summary.

### Stage 3 — Review Results
The results are displayed in a split-screen **Dashboard**:
- **Vision Workspace** (Left) — Shows the original X-Ray with an AI-annotated overlay. Users can toggle the bounding box annotations on/off with an "AI Grounding" switch.
- **Intelligence Workspace** (Right) — A tabbed interface showing:
  - **Raw Clinical Report** — The original findings and impression in a monospaced terminal-style display.
  - **Patient Summary** — A plain-language, 6th-grade reading level summary generated by Gemini, with a medical disclaimer.

## 🔬 Model Training Details

The YOLO model was fine-tuned on **Kaggle** using the **Indiana University Chest X-Ray dataset**, which contains paired radiological images and clinical reports.

### Training Configuration
```python
model = YOLO('yolov8s.pt')  # YOLOv8 Small as base

results = model.train(
    data='bulletproof_medical_data.yaml',
    epochs=25,
    imgsz=512,
    batch=16,
    device=0,         # GPU
    workers=2
)
```

### Detected Anomaly Classes
The fine-tuned model can detect and localize the following medical conditions on Chest X-Rays:
- Cardiomegaly (enlarged heart)
- Aortic Enlargement
- Pleural Effusion
- Pulmonary Fibrosis
- Nodule/Mass
- And other thoracic abnormalities

> Detections labeled as "No finding" are automatically filtered out.

### Training Pipeline
1. **Data Ingestion** — Loaded the Indiana University dataset with `indiana_reports.csv` and `indiana_projections.csv` to pair images with clinical text.
2. **Base Model** — Started with pre-trained `yolov8s.pt` (COCO weights).
3. **Fine-Tuning** — Trained for 25 epochs at 512×512 resolution with a batch size of 16 on Kaggle GPU.
4. **Validation** — Tested on random sample batches, filtering out "No finding" detections.
5. **Export** — The best weights (`best.pt`) were exported and placed in `backend/weights/`.

## 🚀 Getting Started

### Prerequisites
- **Node.js** ≥ 18
- **Python** ≥ 3.10
- **Google API Key** (for Gemini LLM access)

### 1. Clone the Repository
```bash
git clone https://github.com/yourusername/medical_report_analyzer.git
cd medical_report_analyzer
```

### 2. Backend Setup
```bash
cd backend

# Create and activate virtual environment
python -m venv .venv
.venv/Scripts/activate        # Windows
# source .venv/bin/activate   # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
# Create a .env file with your Google API key:
echo "GOOGLE_API_KEY=your_google_api_key_here" > .env

# Run the server
uvicorn app.main:app --reload
```
The API will be available at `http://localhost:8000`.

### 3. Frontend Setup
```bash
cd frontend

# Install dependencies
npm install

# Run the dev server
npm run dev
```
The app will be available at `http://localhost:3000`.

## 🔌 API Reference

### `POST /api/analyze`

Accepts a multipart form with two files and returns a full analysis.

**Request:**
| Field | Type | Description |
|---|---|---|
| `image` | `File` | Chest X-Ray image (PNG/JPG) |
| `report` | `File` | Clinical report (PDF/TXT) |

**Response:**
```json
{
  "annotated_image": "data:image/jpeg;base64,...",
  "summary": "Your X-ray results are ready! The good news is...",
  "detections": [
    {
      "name": "Cardiomegaly",
      "confidence": 0.87,
      "box": [120.5, 200.3, 450.1, 520.7]
    }
  ],
  "raw_report": {
    "findings": "The heart size is enlarged...",
    "impression": "Cardiomegaly and aortic unfolding..."
  }
}
```

## 🌐 Deployment

### Frontend → Vercel
The Next.js frontend is deployed to **Vercel**:
1. Push the `frontend/` directory to a GitHub repository.
2. Connect the repository to Vercel.
3. Set the **Root Directory** to `frontend`.
4. Add the environment variable for the backend API URL.

### Backend → Render
The FastAPI backend is deployed to **Render** using Docker:
1. Push the `backend/` directory to a GitHub repository.
2. Create a new **Web Service** on Render.
3. Set the **Root Directory** to `backend`.
4. Render will automatically detect the `Dockerfile`.
5. Add `GOOGLE_API_KEY` as an environment variable in Render's dashboard.
